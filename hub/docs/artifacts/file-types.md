# Understanding File Types

In the EDA world, many file formats exist. OpenROAD uses a few standardized inputs and an optimized internal database.

## `.lef` and `.def` Files
- **LEF (Library Exchange Format)**: Contains the physical layout information of standard cells and macros. Wait for `read_lef` before reading designs.
- **DEF (Design Exchange Format)**: Contains the exact netlist and placed locations of instances. Version 5.6 is standard in OpenROAD.

## `.odb` (OpenDB)
The OpenDB (`odb`) module in OpenROAD is a design database to support tools for physical chip design. The structure of OpenDB is based on the text file formats LEF and DEF formats version 5.6. 

OpenDB supports a binary file format (`.odb`) to save and load the design much faster than using LEF and DEF text formats. OpenDB is written in C++ 11 with standard library style iterators.

## `.spef`
Standard Parasitic Exchange Format. Generated by `OpenRCX` during the parasitics extraction phase, this file contains the resistance and capacitance of all routed nets.

## `.sdc`
Synopsys Design Constraints. Contains timing constraints like `create_clock` and `set_max_transition`.
