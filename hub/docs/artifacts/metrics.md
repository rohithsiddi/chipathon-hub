# Reading `metrics.json`

During the GSoC and Chipathon evaluation, the `metrics.json` file is one of the most critical artifacts generated by OpenROAD Flow Scripts.

## Final Statistics
OpenROAD automatically calculates area, power, timing, and routing stats after the complete execution of the design flow from RTL-to-GDS. 

Key metrics evaluated by mentors include:
- `wns`: Worst Negative Slack (must be $\ge 0$ for setup and hold to pass timing).
- `tns`: Total Negative Slack. If you have setup violations, this indicates the sum of all failing paths.
- `drc_violations`: Routing DRC violations (must be 0 for final Chipathon tapeout).
- `antenna_violations`: Antenna rule violations (must be 0).
- `core_utilization`: The density of the standard cell placement within the core area.

## Accessing from Python
Because `metrics.json` is a standard JSON file, scripts can parse it easily to build leaderboard dashboards or automate evaluation.
